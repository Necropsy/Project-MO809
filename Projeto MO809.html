Proposta¶

Observando o crescente avano da utilizao de drones para as mais diversas atividades e cenrios, este trabalho prope-se a utilizao de UAVs (Unmanned Aerial Vehicle) para exercer o papel de RSUs mveis, podendo assim acompanhar as variadas demandas que possam surgir. O alto poder de mobilidade destes veculos frente ao poder de processamento pode fornecer um ganho em relao abrangncia ou cobertura de modelos tradicionais de disseminao de dados baseado em estruturas, como podemos citar as RSUs. Deste modo para este problema e projeto de disciplina, ser adotado um modelo de otimizao multiobjetiva, em especfico o algoritmo gentico multiobjetivo NSGA II (Non-Dominated Sorting Genetic Algorithm). O objetivo determinar a taxa mxima de cobertura que pode ser obtida frente a um nmero mnimo de UAVs. Deste modo, a alocao de drones, que exercero o papel das tradicionais RSUs, acompanhar as diferentes transies da mobilidade urbana presente em diferentes zonas/regies.

Construção da Solução¶

Algoritmo NSGA II¶

O modelo evolutivo adotado, como j mencionado anteriormente, foi o algoritmo NSGA, este que um algoritmo evolutivo destinado soluo de problemas multiobjetivos. Proposto por Deb et al. em 2002 [5], uma descrio do funcionamento do algoritmo pode ser obtida a seguir:

Inicialmente criada uma populao de modo aleatrio (P). A partir desta populao gerada uma nova populao de filhos (Q) pelo processo de reproduo e mutao implementados e comum ao modelo tradicional de algoritmos genticos de objetivo nico. A partir da populao constituda de pais e filhos (R) realizado o processo de identificao da frente de pareto (conjunto de timos de pareto), os indivduos presentes nesta frente so considerados aptos a seguirem para a prxima gerao (F1). Diferentemente do modelo proposto em [5], em que o tamanho da populao que deve ser levada para a prxima gerao fixo, aqui foi adaptado para um modelo de intervalo, ou seja, se os indivduos pertencentes a frente pareto for menor que um limitante inferior estes so separado e o processo de identificao da frente pareto realizado novamente no grupo restante, gerando novos grupos (F2 e F3). Porm, caso o nmero de indivduos selecionados for acima que um limitante superior realizado um processo de seleo por agrupamento para remover os excedentes (Rejected). Se o nmero de indivduos selecionados pelas frentes de pareto estiverem entre estes limitantes avanamos para a prxima gerao (new P), todo o processo se repete enquanto no for atingido o nmero mximo de iteraes ou outro critrio de parada for identificado. Por fim, quando este lao de repetio interrompido realizado novamente a identificao de uma frente pareto, desta forma so selecionados os melhores indivduos dentre toda a populao.

OBS: A utilizao do limite superior e inferior, utilizados para determinao dos indivduos aptos para prxima gerao, permite uma maior variabilidade de solues dentro de uma mesma populao. Pois tais indivduos passam a no necessariamente pertencerem a mesma frente de pareto.

Mais detalhes referentes a cada etapa descrita acima podem ser obtidos nas sees de suas respectivas implementaes.

Alguns trabalhos na literatura abordam a utilizao de uma representao simplificada da topologia das rodovias. O processo consiste em segmentar o mapa em clulas adjacentes e de mesma dimenso (representao em grid). Os benefcios em se utilizar este modelo de representao so:

  • Controle de preciso;
  • Complexidade computacional;
  • Complexidade de soluo.

Este modelo nos permite um maior controle sobre a preciso exigida no problema, ou seja, com a variao do tamanho das clulas possvel obter uma variao no nvel de detalhes inerentes ao cenrio. Desta forma, a exigncia computacional tambm pode ser reduzida, tendo em vista que a complexidade da soluo funciona do mesmo modo para representaes de diferentes nveis de detalhes.

Foram utilizadas duas representaes de malhas virias:

  • Mapa 01: 5 Km
  • Mapa 02: 2 Km

Em ambos os mapas cada clula representam 100m, deste modo, no Mapa_01 temos um grid 50x50 e no Mapa_02 um grid 20x20.

Eficiência de Pareto¶

Para utilizarmos o algoritmo gentico com mltiplos objetivos necessrio adotar um modelo de anlise conjunto. Uma soluo a utilizao da eficincia de pareto ou frente de pareto, esta abordagem define uma fronteira baseada no compromisso entre os mltiplos objetivos, este compromisso est definido pelo fato de no ser possvel aprimorar um sem piorar os demais. As funes a seguir so utilizadas para identificar e armazenar as frentes de pareto.

#PARETO
def identify_pareto(scores, population_ids):

    population_size = scores.shape[0]
    pareto_front = np.ones(population_size, dtype=bool)
    for i in range(population_size):
        for j in range(population_size):
            if all(scores[j] >= scores[i]) and any(scores[j] > scores[i]):
                pareto_front[i] = 0
                break

    return population_ids[pareto_front]

def build_pareto_population(population, scores, minimum_population_size, maximum_population_size):
  
    unselected_population_ids = np.arange(population.shape[0])
    all_population_ids = np.arange(population.shape[0])
    pareto_front = []
  
    while len(pareto_front) < minimum_population_size:

        temp_pareto_front = identify_pareto(scores[unselected_population_ids, :], unselected_population_ids)
        combined_pareto_size = len(pareto_front) + len(temp_pareto_front)
    
        if combined_pareto_size > maximum_population_size:
            number_to_select = combined_pareto_size - maximum_population_size
            selected_individuals = (reduce_by_crowding(scores[temp_pareto_front], number_to_select))
            temp_pareto_front = temp_pareto_front[selected_individuals]
        
        pareto_front = np.hstack((pareto_front, temp_pareto_front))

        unselected_set = set(all_population_ids) - set(pareto_front)
        unselected_population_ids = np.array(list(unselected_set))

    population = population[pareto_front.astype(int)]

    return population

Agrupamento e Torneio¶

Como estamos utilizando a frente de pareto no possvel fazer uma seleo simplificada de indivduos que passaro para a prxima gerao, um exemplo de seleo simplificado seria rankear cada soluo e eliminar os com menor aptido, porm como todos os indivduos pertencem a uma frente pareto, no existe um indivduo que se sobressai aos demais. Deste modo, para realizar a seleo das geraes foi utilizado o mtodo de agrupamento junto a um processo de torneio.

Exemplificando, realizado o processo de agrupamento baseando-se na distncia entre os vizinhos de cada indivduo. Aps este processo d-se incio a etapa de torneio, onde so selecionados 2 indivduos de modo aleatrio e so comparadas suas distncias de agrupamento, os que apresentarem maior valor de distncia so selecionados para a prxima etapa.

def crowding(scores):

    pop_size = len(scores[:,0])
    num_score = len(scores[0,:])
  
    matrix_crowding = np.zeros((pop_size, num_score))
    #normalizando os scores
    normed_scores = ( scores-scores.min(0) ) / scores.ptp(0)

    for col in range(num_score):
        crowding = np.zeros(pop_size)

        crowding[0] = 1
        crowding[pop_size - 1] = 1

        sorted_scores = np.sort(normed_scores[:, col])
        sorted_scores_index = np.argsort(normed_scores[:, col])
        crowding[1:pop_size - 1] = (sorted_scores[2:pop_size] - sorted_scores[0:pop_size - 2])
        re_sort_order = np.argsort(sorted_scores_index)
        sorted_crowding = crowding[re_sort_order]
        matrix_crowding[:, col] = sorted_crowding

    crowding_distances = np.sum(matrix_crowding, axis=1)

    return crowding_distances

#Torneio
def reduce_by_crowding(scores, number_to_select):
  
    population_ids = np.arange(scores.shape[0])
    crowding_distances = crowding(scores)
    picked_population_ids = np.zeros((number_to_select))
    picked_scores = np.zeros((number_to_select, len(scores[0, :])))

    for i in range(number_to_select):

        population_size = population_ids.shape[0]
        fighter1ID = rd.randint(0, population_size - 1)
        fighter2ID = rd.randint(0, population_size - 1)

        if crowding_distances[fighter1ID] >= crowding_distances[fighter2ID]:
            picked_population_ids[i] = population_ids[fighter1ID]
            picked_scores[i, :] = scores[fighter1ID, :]
            population_ids = np.delete(population_ids, (fighter1ID),axis=0)
            scores = np.delete(scores, (fighter1ID), axis=0)
            crowding_distances = np.delete(crowding_distances, (fighter1ID),axis=0)
        else:
            picked_population_ids[i] = population_ids[fighter2ID]
            picked_scores[i, :] = scores[fighter2ID, :]
            population_ids = np.delete(population_ids, (fighter2ID), axis=0)
            scores = np.delete(scores, (fighter2ID), axis=0)
            crowding_distances = np.delete(crowding_distances, (fighter2ID), axis=0)

    picked_population_ids = np.asarray(picked_population_ids, dtype=int)

    return (picked_population_ids)

Parâmetros de Controle¶

Estes so os parmetros de controle de execuo do algoritmo:

  • map_size, tamanho do mapa que est sendo utilizado;
  • max_dim, dimenses da matriz obtida atravs do mapa;
  • road_units, este parmetro informa quantas unidades de pista existem no mapa (utilizado para otimizao do algoritmo);
  • uav_range, raio de alcance dos drones;
  • chromosome_size, tamanho do cromossomo ou indivduo, tambm representa o nmero mximo de drones disponveis para construo de uma soluo;
  • prob_mutation, probabilidade de um indivduo sofrer mutao;
  • starting_population_size, populao inicial gerada aleatoriamente;
  • maximum_generation, nmero mximo de geraes que sero executadas caso no haja ou no ocorra um critrio de parada;
  • minimum_population_size, limite mnimo de indivduos que passaro para a proxima gerao;
  • maximum_population_size, limite mximo de indivduos que passaro para a proxima gerao;
  • time_execution, critrio de parada baseado no tempo mximo de execuo, caso o valor atribuido a este parmetro for zero, o algortmo no considera o tempo de execuo.

OBS: A necessidade de alguns parmetros foi observada durante o perodo de testes do algoritmo, por exemplo, o critrio de para time_execution foi estipulado ao ocorrerem execues em que, dado mapas maiores e mais complexos, o algoritmo gastou aproximadamente 3 horas de execuo, fator que pode tornar a aplicao invivel dependendo do objetivo ao qual forem utiliz-la.

Conclusões¶

OBS: Os parmetros utilizado em cada teste descrito em sua respectiva seo.

Tamanho da população¶

Para a avaliao do impacto do tamanho da populao nas solues obtidas, foram realizados dois testes, o primeiro utilizou uma populao inicial de 2000 indivduos e com um intervalo entre o limite superior de 450 e inferior de 50 (figura 1). No segundo teste os limites utilizados foram, superior igual a 750 e inferior de 100 (figura 2). Durante cada gerao a populao flutua dentro destes intervalos, estes fator no permite um controle preciso sobre o nmero de indivduos que passaro para a prxima gerao porm permitem uma maior variabilidade entre os mesmos. Ao analisar os dados obtidos no foi possvel observar grandes diferenas nas solues, porm podemos destacar o tempo de execuo, para a primeira execuo o algoritmo levou cerca de 8 minutos e j no segundo teste o tempo aproximado foi de 13 minutos, Um acrscimo considervel quando analisado o acrscimo do intervalo entre os limites.

Critérios de parada¶

Na anlise dos critrios de parada, a figura 1 expressa uma execuo completa do algoritmo, ou seja, o algoritmo realizou todas as geraes estipuladas, em seguida o algoritmo foi limitado pelo tempo de execuo, deste modo ao atingir um tempo especfico o algoritmo retorna a melhor soluo j encontrada. As figuras 3 e 4 representam as solues obtidas com as restries 4 min. e 2 min. respectivamente. possvel notar a diferena na qualidade das solues obtidas, na restrio de 4 min. foi encontrado um indivduo/soluo que utiliza apenas 11 drones e apresenta uma taxa de cobertura da topologia de aproximadamente 72%, enquanto que para a execuo limitada a 2 min. para a obteno de uma taxa de cobertura similar foi necessrio a utilizao de 14 drones.

Técnica de crossover¶

Um dos resultados que mais chamaram a ateno nos testes est associado ao modelo de execuo da tcnica de crossover. Como j descrito anteriormente, foram implementados 2 modelos de reproduo baseados na tcnica de crossover, a primeira implementao realiza e baseada em um ponto de corte obtido de modo aleatrio, desta forma existe a oportunidade da gerao de uma maior variabilidade entre os prprios indivduos de uma mesma gerao. J na segunda implementao utiliza-se 2 pontos de corte fixos, onde as 3 sees geradas pelos cortes correspondem a 1/3 de cada gene, sendo combinados alternadamente entre o indivduo A e B. A figura 5 expressa os dados obtidos ao utilizar a primeira tcnica e a figura 6 est relacionada a utilizao do segundo modelo. Ambas as execues rodaram sobre os mesmo parmetros, porm mesmo considerando a aleatoriedade presente nos algoritmos genticos, aps diversas execues durante a construo deste projeto a discrepncia entre as solues obtidas se manteve em sua grande maioria. Como observado, o primeiro modelo indica a construo de populaes com maior variabilidade entre seus indivduos, diferentemente da segunda abordagem.

Técnica de mutação¶

Os resultados obtidos entre as tcnicas de mutao apresentaram o comportamento esperado frente a proposta de cada uma das abordagens utilizadas. Como a primeira implementao executa a remoo de drones da soluo, foi possvel observar que os resultados obtidos (figura 7) ao utiliz-la obtiveram maior eficcia na utilizao deste objetivo em especfico, ou seja, a amplitude das solues foi maior do que os obtidos pela outra abordagem que implementa a troca da posio dos drones por novos locais disponveis. Porm, mesmo apresentando menor abrangncia, as solues encontradas (figura 8) tambm apresentam boa correlao entre a taxa de cobertura e o numero de drones. A abrangncia da primeira abordagem indica que esta permite uma varredura mais ampla sobre o espao de busca.

Taxa de mutação¶

Como mencionado, foi realizado 2 baterias de testes para a anlise da taxa de mutao, na primeira bateria foi utilizada o tipo 1 de mutao e para a segunda bateria foi adotado o 2 modelos de mutao. Em cada bateria so realizadas 5 execues com taxas distintas, sendo elas: 43% (figura 9, 14), 6% (figura 10, 15), 18% (figura 11, 16), 94% (figura 12, 17) e 28% (figura 13, 18).

Modelo de mutao tipo 1:

Nos primeiro resultados obtidos foi possvel observar que a taxa de mutao de 94% (figura 12) no forneceu maior variabilidade e amplitude entre as solues como o esperado, por exemplo, com a taxa de mutao em 6% (figura 10) os resultados obtidos foram melhores, porm em outras execues realizadas durante o processo de construo deste projeto este comportamento no se manteve, onde para taxas maiores a amplitude das solues (mais solues com diferentes relacionamentos entre os objetivos) apresentou-se maior.

Modelo de mutao tipo 2:

Na segunda bateria de testes foi observado que as solues obtidas mantiveram-se prximas, com destaque para a taxa de 18% (figura 16) que conseguiu fornecer um melhor alcance de taxa de cobertura mediante a utilizao do nmero de drones utilizados (porm estes valores ainda prximos de outras solues obtidas em mais taxas de mutao).

Todos os testes realizados nesta seco forneceram uma viso sobre o impacto de cada parmetro que constituem o algoritmo, testes adicionais foram realizados ao longo da construo deste projeto e serviram como base de argumentao em diversas sees. Devo ressaltar que alguns tpicos como tcnica de seleo, mtodo de substituio e taxa de crossover no se aplicam a este projeto pelo fato do algoritmo implementado no apresentar controle especfico sobre estes fatores.
